### T3 — Database schema (Supabase SQL) ⚠️ Medium
Create tables with RLS policies.

**SQL to produce:** `backend/supabase/schema.sql`

```sql
-- posts table
create table posts (
  id uuid primary key default gen_random_uuid(),
  author_id uuid references auth.users not null,
  title text not null,
  slug text unique not null,
  content text not null,
  excerpt text,
  cover_image_url text,
  status text not null default 'draft' check (status in ('draft','published')),
  category text,
  tags text[] default '{}',
  search_vector tsvector,
  reading_time_minutes int,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- categories table (lookup)
create table categories (
  id uuid primary key default gen_random_uuid(),
  name text unique not null,
  slug text unique not null
);

-- RLS
alter table posts enable row level security;

-- Public can read published posts
create policy "public_read_published" on posts
  for select using (status = 'published');

-- Authors read own drafts
create policy "author_read_own" on posts
  for select using (auth.uid() = author_id);

-- Authors insert own posts
create policy "author_insert" on posts
  for insert with check (auth.uid() = author_id);

-- Authors update own posts
create policy "author_update" on posts
  for update using (auth.uid() = author_id);

-- Authors delete own posts
create policy "author_delete" on posts
  for delete using (auth.uid() = author_id);

-- tsvector trigger
create or replace function update_search_vector()
returns trigger as $$
begin
  new.search_vector := to_tsvector('english', coalesce(new.title,'') || ' ' || coalesce(new.content,''));
  return new;
end;
$$ language plpgsql;

create trigger posts_search_vector_update
  before insert or update on posts
  for each row execute function update_search_vector();
```

**Acceptance:** Schema applies without error; RLS blocks anonymous reads of drafts.

### T4 — Frontend scaffold ✅ Simple
Create `frontend/` with Vite + React + TypeScript + Tailwind + shadcn/ui.

**Steps:**
```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
npm install @supabase/supabase-js
npm install lucide-react
npx shadcn@latest init   # choose: New York style, Slate, yes CSS variables
```

**Files to create/modify:**
- `frontend/src/lib/supabase.ts` — Supabase client from `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY`
- `frontend/src/types/index.ts` — `Post`, `User`, `AuthRole` TypeScript types
- `frontend/src/App.tsx` — minimal router shell (just a health check page for now)
- `frontend/.env.local` — copy from `.env.example` VITE_* vars (gitignored)
- `frontend/.gitignore` — node_modules/, dist/, .env.local

**Acceptance:** `npm run dev` starts on port 5173; `npm run build` exits 0; `npm run lint` exits 0.

### T5 — Frontend auth context ✅ Simple

**Files to create:**

**Acceptance:** `useAuth()` returns `role: 'author'` for author test user, `role: 'reader'` for reader test user, `null` for unauthenticated.
