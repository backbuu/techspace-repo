4. **Supabase schema — Phase 1 tables**

```sql
-- Profiles (extended user info)
create table profiles (
  id uuid primary key references auth.users on delete cascade,
  username text unique,
  bio text,
  avatar_url text,
  role text not null default 'reader' check (role in ('reader', 'author')),
  created_at timestamptz default now()
);
alter table profiles enable row level security;
create policy "public read" on profiles for select using (true);
create policy "own write" on profiles for all using (auth.uid() = id);

-- Posts
create table posts (
  id uuid primary key default gen_random_uuid(),
  author_id uuid references auth.users not null,
  title text not null,
  slug text unique not null,
  excerpt text,
  content text not null,            -- raw markdown
  cover_image_url text,
  status text not null default 'draft' check (status in ('draft', 'published')),
  reading_time_minutes int,
  published_at timestamptz,
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  search_vector tsvector             -- populated by trigger
);
alter table posts enable row level security;
-- Public can read published posts
create policy "public read published" on posts for select using (status = 'published');
-- Authors can read/write their own posts
create policy "author own posts" on posts for all using (auth.uid() = author_id);

-- Auto-update search_vector and updated_at
create or replace function posts_search_vector_update() returns trigger as $$
begin
  new.search_vector := to_tsvector('english', coalesce(new.title,'') || ' ' || coalesce(new.excerpt,'') || ' ' || coalesce(new.content,''));
  new.updated_at := now();
  return new;
end;
$$ language plpgsql;
create trigger posts_search_vector_trigger
  before insert or update on posts
  for each row execute function posts_search_vector_update();
create index posts_search_idx on posts using gin(search_vector);

-- Tags
create table tags (
  id uuid primary key default gen_random_uuid(),
  name text unique not null,
  slug text unique not null
);
alter table tags enable row level security;
create policy "public read" on tags for select using (true);

-- Post ↔ Tag junction
create table post_tags (
  post_id uuid references posts on delete cascade,
  tag_id uuid references tags on delete cascade,
  primary key (post_id, tag_id)
);
alter table post_tags enable row level security;
create policy "public read" on post_tags for select using (true);
create policy "author write" on post_tags for all
  using (exists (select 1 from posts where id = post_id and author_id = auth.uid()));

-- Categories (single per post)
create table categories (
  id uuid primary key default gen_random_uuid(),
  name text unique not null,
  slug text unique not null,
  description text
);
alter table categories enable row level security;
create policy "public read" on categories for select using (true);

alter table posts add column category_id uuid references categories;

-- Bookmarks
create table bookmarks (
  user_id uuid references auth.users on delete cascade,
  post_id uuid references posts on delete cascade,
  created_at timestamptz default now(),
  primary key (user_id, post_id)
);
alter table bookmarks enable row level security;
create policy "own bookmarks" on bookmarks for all using (auth.uid() = user_id);

-- Reading history
create table reading_history (
  user_id uuid references auth.users on delete cascade,
  post_id uuid references posts on delete cascade,
  last_read_at timestamptz default now(),
  primary key (user_id, post_id)
);
alter table reading_history enable row level security;
create policy "own history" on reading_history for all using (auth.uid() = user_id);
```

   - Validate: tables visible in Supabase dashboard; RLS policies active

## Module 2: Article Listing + Reading

⚠️ **Medium** → sub-plan: `2.article-listing.md`

**Goal:** Public article list with filters; post detail page with markdown rendering, TOC, reading time.

### Tasks

1. **Backend: posts router**
   - `GET /posts` — list published posts; query params: `?category=`, `?tag=`, `?limit=`, `?offset=`
   - `GET /posts/:slug` — single post with tags and category
   - Validate: `curl http://localhost:8000/posts` → JSON array of published posts

2. **Frontend: HomePage**
   - `PostCard` component: cover image, title, excerpt, tags, reading time, author
   - `PostList` with infinite scroll or pagination
   - Category sidebar / tag filter
   - Validate: home page shows article cards; clicking a tag filters the list

3. **Frontend: PostDetailPage**
   - Render `content` markdown via `react-markdown` + `remark-gfm` + `rehype-highlight`
   - Auto-generate Table of Contents from `##` and `###` headings
   - Display reading time estimate
   - Validate: navigate to `/posts/:slug` → markdown renders with syntax highlighting; TOC scrolls to section

4. **Reading time trigger**
   - Backend or DB trigger: on insert/update, calculate `ceil(word_count / 200)` minutes
   - Validate: create a post → `reading_time_minutes` populated

## Module 3: Author CMS

⚠️ **Medium** → sub-plan: `3.author-cms.md`

**Goal:** Author-only create/edit/delete posts, cover image upload, draft vs published toggle.

### Tasks

1. **Backend: author routes (auth required)**
   - `POST /posts` — create post (author only; `author_id` from JWT)
   - `PUT /posts/:id` — update own post
   - `DELETE /posts/:id` — delete own post
   - `POST /posts/:id/publish` — toggle `status = 'published'`, set `published_at`
   - Validate: POST without auth → 401; POST with author JWT → 201

2. **Backend: image upload**
   - `POST /posts/:id/cover` — upload image to Supabase Storage bucket `covers` (public); update `cover_image_url`
   - Validate: upload a JPEG → URL returned; image accessible at public URL

3. **Frontend: Author dashboard**
   - Route `/dashboard` (author role required)
   - Post list with edit/delete/publish actions
   - Markdown editor (textarea + live preview split pane)
   - Cover image uploader (drag-and-drop)
   - Tag and category selectors
   - Validate: author creates post → appears in dashboard as draft; publish → visible on home page

4. **Role guard**
   - Frontend route guard: redirect non-authors away from `/dashboard`
   - Backend middleware: reject non-authors on write endpoints
   - Validate: reader JWT on `POST /posts` → 403

## Module 4: Search

✅ **Simple** → sub-plan: `4.search.md`

**Goal:** Full-text search using Supabase tsvector; highlighted excerpts; filter by tag/category.

### Tasks

1. **Backend: search endpoint**
   - `GET /search?q=&tag=&category=` — query `posts.search_vector @@ plainto_tsquery('english', $q)`, return ranked results with `ts_headline` excerpts
   - Validate: `curl "http://localhost:8000/search?q=kubernetes"` → matching posts with excerpts

2. **Frontend: SearchPage**
   - Search bar (debounced, 300 ms)
   - Results list with highlighted excerpts
   - Tag/category filter chips
   - Validate: type a query → results update; filter narrows results

## Module 5: Reader Features

✅ **Simple** → sub-plan: `5.reader-features.md`

**Goal:** Bookmarks and reading history for authenticated readers.

### Tasks

1. **Bookmarks**
   - Frontend calls Supabase directly (anon key + JWT) — no backend needed (RLS handles it)
   - `BookmarkButton` on `PostDetailPage`: toggle bookmark
   - `BookmarksPage` at `/bookmarks`: list bookmarked posts

   - Validate: read two posts → both appear in history in order

## Module 6: Semantic Search + Related Posts

⚠️ **Medium** → sub-plan: `6.semantic-search.md`

**Goal:** pgvector embeddings on article content; semantic search endpoint; "related posts" widget.

### Tasks

1. Add `embedding vector(1536)` column to `posts`
2. On publish/update: embed `title + excerpt + content` via OpenAI `text-embedding-3-small`
3. `GET /search/semantic?q=` — embed query, cosine similarity search
4. `GET /posts/:slug/related` — return top-5 similar posts by embedding
5. Frontend: `RelatedPosts` widget at bottom of `PostDetailPage`
6. Validate: semantic query on a topic → relevant posts returned even without exact keyword match

## Module 7: AI Writing Assistant

🔴 **Complex** → sub-plan: `7.ai-assistant.md`
